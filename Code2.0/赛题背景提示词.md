1. **赛题背景**

   你正在参与中国大学生服务外包创新创业大赛 A04 企业命题。
   该赛题由教育行业企业提出，背景是：当前 AI 教学工具功能碎片化，
   教师需要在多个工具之间切换完成备课，AI 对教学意图理解浅层，
   无法围绕“教学思路”形成连续、闭环的工作流。

   本赛题希望借助大语言模型与多模态 AI 技术，
   构建一个能通过多轮对话深度理解教师教学设计意图，
   并自动生成、迭代优化教学课件的智能教学辅助系统。

2. **核心目标**

   本项目的核心目标是设计并实现一个“多模态 AI 互动式教学智能体”，
   该智能体应以教师的教学思路为驱动，而非以单次指令或模板为中心。

   系统需要能够：
   1. 通过自然语言（文字/语音）进行多轮对话，逐步澄清并确认教学意图；
   2. 理解并融合教师上传的多模态参考资料（PDF、Word、PPT、图片、视频等）；
   3. 自动生成结构完整、内容合理的 PPT 课件与 Word 教案初稿；
   4. 支持教师基于预览结果进行反馈修改，形成可迭代优化的闭环流程。

3. **功能性要求**

   - 智能体需要具备教学意图理解能力，能够从教师的多轮对话中，
     结构化提取以下教学要素：

     - 教学目标
     - 核心知识点及逻辑顺序
     - 重点与难点
     - 教学时长
     - 课堂互动或教学风格偏好

     当教师描述不完整或模糊时，系统应能主动提问以澄清需求，
     并在关键节点对已理解内容进行总结确认。

   - 系统需支持教师上传多种格式的参考资料，包括但不限于：
     PDF、Word、PPT、图片、视频。

     智能体需要能够：
     - 从文档中提取文本结构、知识点与排版风格；
     - 对视频进行关键帧分析或内容摘要；
     - 理解“参考哪一部分内容 / 哪种风格”的对应关系，
     并将解析结果用于后续课件生成。

   - 系统需构建一个本地知识库，并基于检索增强生成（RAG）机制使用：

     - 教师专业相关资料可以作为知识库输入；
     - 对文本进行向量化、检索和上下文注入；
     - 在课件生成阶段，将本地知识库内容与教师意图、参考资料进行融合，
     以提升内容准确性与专业性。
     - 教师可以上传在对话时文档，对其做暂存处理，并和知识库一起作为用户对话时检索的资料，返回切分后的文段作为传给LLM生成答案的参考

   - 系统需要支持以下课件生成能力：

     1. PPT 自动生成：
        - 包含封面、目录、内容页、总结页
        - 内容图文并茂，逻辑清晰
     2. Word 教案生成：
        - 教学目标
        - 教学过程
        - 教学方法
        - 课堂活动设计
        - 课后作业
     3. 内容多样性：
        - 至少支持一种：动画创意 / 互动小游戏 / 创新教学形式

   - 在课件生成后，系统需提供预览与反馈能力：

     - 教师可以提出自然语言修改意见（如调整顺序、简化内容、补充案例）；
     - 智能体需理解修改意图并重新生成课件；
     - 支持最终成果以标准格式导出（.pptx, .docx）；
     - 动画或互动内容可通过 HTML5 / GIF / MP4 等形式导出或集成。

4. **技术约束**

   项目在技术实现上需满足以下基本约束：

   - 至少使用一个大语言模型，作为对话、意图理解与内容生成的核心；
   - 实现本地知识库与 RAG 机制；
   - 至少展示两种不同格式的多模态资料解析能力（如 PDF + 视频）；
   - 可基于开源大模型和相关算法进行开发；
   - 强调实用性、完整流程与创新性，而非单点功能演示。
